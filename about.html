features

Available Offline

Allume uses HTML5's application cache to make it available offline. Just visit allume.cc once, and you are ready to go! On node.js, just install allume from the npm repository.

Custom Repositories

You can specify custom public repositories. At this point, we are investigating how to implement private, secure repositories. Help welcome!

Cross-Runtime

Allume is designed to run inside browsers and node.js. It should work on any browser that supports HTML5 features or any runtime that supports the node.js api.

usage

1. create

Create a package to run on multiple runtime environments, such as a browser or node.js. Allume can load pkx packages from http and the file-system (in supported runtime environments). With pkx you create a tar file with ".pkx" extension of your app or library that has a package.json file, and publish it to a public repository (which is just a static http server). In future versions of pkx, offline caching of the packages will make sure your app or library can be used offline. Allume already works offline by using HTML5's application cache. You can also install allume locally from npm for node.js. 

                        
    (function() {
        function Demo(pkx, module, config) {
            var self = this;

            this.sayHello = function(to) {
                console.log("Hello, " + to + "!");
            }

            self.sayHello(config? config.name : "guest");
        }

        define(function() {
            return
                new (Function.prototype.bind.apply(Demo, arguments));
        });
    })();

2. deploy

Upload the pkx package to a public webserver.

3. run

node.js

$ allume cc.demo --config "{ 'name': 'Dave' }" --repo http://demo.com_

Hello, Dave!_
